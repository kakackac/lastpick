# Data and Government Strategy

## Data Types

| Type | Example | Access |
|---|---|---|
| Seller input data | category, quantity, price, pickup time | MVP에서 직접 수집 |
| Platform data | orders, pickup rate, reviews, repeat purchase | 서비스 운영으로 축적 |
| Public data | weather, population, commercial area, food safety | 공공데이터포털 등 |
| Enterprise data | POS, inventory, expiration, sales | 본사/점주/POS 협약 필요 |

## Public Data Use

공공데이터는 편의점 내부 DB와 다릅니다. 공공데이터 경진대회에서는 내부 재고 데이터를 받는 것이 아니라, 상권/날씨/유동인구/식품안전 데이터를 활용해 수요예측과 추천 로직을 설계하는 방향이 적합합니다.

## Competition Topic

공공데이터와 점포 입력 데이터를 결합해 동네 신선식품의 마감 판매 가능성을 예측하고, 소비자에게 근거리 픽업 상품을 추천하는 푸드세이빙 플랫폼.

## Government Programs to Track

- 민관협력 오픈이노베이션: 대기업/마트/편의점 PoC
- 환경분야 청년창업 지원사업: 식품폐기 감축
- ESG 지원사업: 폐기 절감, 탄소 감축
- 공공데이터 경진대회: 수요예측/추천 로직 검증
- 데이터바우처: 데이터 구매, 가공, AI 분석 비용 지원
- AI/PoC 지원사업: 할인율 추천, 폐기율 예측 모델 개발

## Enterprise Data Roadmap

1. Seller manual input
2. Store templates and repeat registration
3. POS CSV import
4. POS vendor partnership
5. Franchise/chain PoC
6. Convenience store headquarters API integration
